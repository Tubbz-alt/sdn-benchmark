Overview

SRI's Northbound API has two components, (1) a floodlight-specific server module and (2) language-specific client libraries. Clients communicate to the server via predefined (but specifiable) TCP ports using Google Protocol Buffer messages.

Server Configuration

The server settings are documented in

  • AbstractNBListener, for plain communications, and
  • NBServerSSLSupport, for SSH-based communications (see SecureCommunications for certificate generation).

Note that the common prefix for the runtime/config/floodlight.properties keys is

    net.floodlightcontroller.security.northboundapi.server.SENorthboundAPI.

Client Libraries

This distribution contains Java and Perl Northbound API client libraries, as well as the Google Protocol Buffer specifications upon which these libraries depend.

Java

For Java, the library is

    dev/java/ClientNBAPI.jar

where OpenFlow.NBAPI.ControllerContext provides core library functionality.

Perl

For Perl, the modules reside in

    dev/perl

and OpenFlow::NBAPI::ControllerContext provides core library functionality.

Google Protocol Buffer Specifications

Finally, the Google Protocol Buffer specifications, which employ protobuf-2.5.0 syntax, reside in

    dev/proto

where NorthboundAPIPB.proto:MessagePB is the root data structure.
